# Empty State Pattern

## When To Use

- When a list, detail area, or workflow has no user data yet.

## Why It Works

- It explains the gap and gives the user a confident first step.

## What To Preserve

- One-sentence context.
- One clear action.
- The same visual language as the rest of the app.

## What To Avoid

- Long explanatory paragraphs.
- Multiple competing next steps.
