# Dashboard Pattern

## When To Use

- Only when a user genuinely needs a quick summary plus one clear next step.

## Why It Works

- It can orient the user quickly without forcing them through multiple pages.

## What To Preserve

- A single primary action.
- Minimal supporting metrics.
- Clear grouping, not a wall of cards.

## What To Avoid

- Generic admin dashboards.
- Status blocks that do not change the next action.
