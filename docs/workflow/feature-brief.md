# Feature Brief

Use this before non-trivial implementation.

## GitHub Issues

- When working from a GitHub issue, automatically create a lightweight brief for medium or large issues.
- Skip the brief for trivial issues such as small copy edits, narrow bug fixes, or mechanical cleanup.
- Keep issue-based briefs concise and decision-focused.

## Capture

- User goal: what the user is trying to accomplish.
- UX flow: the shortest path from entry to success.
- Primary action: the one thing the screen or feature should make obvious.
- Architecture plan: affected routes, components, data, and tests.
- Risks: auth, ownership, migration, performance, and UX risks.
- Simplification pass: what can be removed, delayed, or merged.
- Acceptance criteria: observable behavior that proves the work is done.

Keep the brief short. If it takes too long to write, it is too heavy.
