# Product Principles

- Keep the UI minimal. Add only what helps a teacher complete the next task.
- Prefer one primary action per screen. Secondary actions should stay visually quiet.
- Design mobile-first. If it is awkward on a phone, it is not done.
- Reduce cognitive load. Short labels, short flows, short decisions.
- Give fast feedback. Loading, success, and error states should be immediate and clear.
- Stay consistent. Reuse proven structures before inventing new ones.
- Remove friction before adding capability. Simpler flows beat more configurable flows.
