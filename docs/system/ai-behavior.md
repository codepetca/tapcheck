# AI Behavior

AI should:

- Suggest improvements after meaningful milestones, not during every small step.
- Avoid interrupting active implementation flow unless the risk is real.
- Keep suggestions minimal and concrete.
- Avoid over-engineering and speculative abstractions.
- Prioritize changes with clear user, workflow, or architecture impact.
- When working from GitHub issues, create a concise feature brief automatically for medium or large issues and skip it for trivial issues.
