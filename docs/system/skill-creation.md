# Skill Creation

Suggest a new skill only when at least one of these is true:

- The same workflow has appeared 3 or more times.
- The work is clearly multi-step and repeatable.
- There is a good automation opportunity.
- A reusable product or architecture pattern is emerging.

Good examples:

- `feature-brief`
- `critique-ui`
- `post-pr-review`
- `architecture-plan`
- `extract-pattern`

Rules:

- Suggest skills, do not create them automatically.
- Ask for approval before creating or installing a skill.
- Avoid excessive skill sprawl. A short doc or accepted pattern is often enough.
