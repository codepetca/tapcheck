# Self-Improving Loop

Run this lightweight loop after:

- a PR merge
- a feature completion
- repeated friction in the same workflow
- an architecture decision that is likely to recur

Process:

1. Review the change or friction point.
2. Identify reusable patterns.
3. Suggest a small improvement.
4. Ask for approval before changing shared docs or workflows.
5. Update the relevant docs after approval.

Keep the loop small. The goal is better defaults, not more process.
